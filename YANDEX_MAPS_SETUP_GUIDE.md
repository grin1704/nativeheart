# Руководство по настройке Яндекс.Карт

## Что такое "Место захоронения"

Функция позволяет:
- 📍 Указать адрес места захоронения
- 🗺️ Показать его на карте
- 📝 Добавить описание и инструкции как найти могилу
- 🔍 Автоматически определить координаты по адресу

## Текущий статус

### ✅ Backend - Полностью реализован
- API endpoints для CRUD операций
- Автоматическая геокодировка адресов
- Интеграция с Яндекс.Геокодером
- Хранение координат и адресов
- Проверка доступа (владелец/коллаборатор)

### ⚠️ Frontend - Частично реализован
- Компоненты созданы
- Требуется подключение Яндекс.Карт API
- Требуется настройка API ключей

## Что нужно для работы

### 1. API ключи Яндекс

Вам нужны **два** API ключа:

#### A. Yandex Maps API Key
**Для чего:** Отображение карт на frontend

**Как получить:**
1. Перейдите на https://developer.tech.yandex.ru/
2. Войдите или зарегистрируйтесь
3. Перейдите в раздел "JavaScript API и HTTP Геокодер"
4. Нажмите "Получить ключ"
5. Выберите "JavaScript API"
6. Скопируйте ключ

#### B. Yandex Geocoder API Key
**Для чего:** Преобразование адресов в координаты на backend

**Как получить:**
1. На том же сайте https://developer.tech.yandex.ru/
2. Получите ключ для "HTTP Геокодер"
3. Скопируйте ключ

### 2. Настройка Backend

**Файл:** `backend/.env`

Добавьте ключи:
```env
# Yandex Maps API (для frontend)
YANDEX_MAPS_API_KEY=ваш-ключ-maps-api

# Yandex Geocoder API (для backend геокодирования)
YANDEX_GEOCODER_API_KEY=ваш-ключ-geocoder-api
```

**Примечание:** Без этих ключей:
- ❌ Карты не будут отображаться
- ❌ Автоматическое определение координат не будет работать
- ✅ Но можно вручную вводить координаты

### 3. Настройка Frontend

**Файл:** `frontend/.env.local` (создайте если нет)

```env
NEXT_PUBLIC_YANDEX_MAPS_API_KEY=ваш-ключ-maps-api
```

**Важно:** Префикс `NEXT_PUBLIC_` обязателен для доступа в браузере!

## Как это работает

### Для владельца страницы

#### 1. Открыть редактор страницы
```
http://localhost:3000/dashboard/edit/[page-id]
```

#### 2. Перейти на вкладку "Место захоронения"

#### 3. Заполнить форму
- **Адрес** (обязательно): "Москва, Новодевичье кладбище, участок 2"
- **Описание** (опционально): "Рядом с часовней"
- **Инструкции** (опционально): "От главного входа прямо 100м, затем налево"

#### 4. Система автоматически:
- Определит координаты по адресу
- Покажет место на карте
- Сохранит данные

### Для посетителей

На публичной странице будет отображаться:
- 📍 Адрес места захоронения
- 🗺️ Интерактивная карта с маркером
- 📝 Описание и инструкции
- 🧭 Кнопка "Построить маршрут" (откроет Яндекс.Карты)

## API Endpoints

### Создать/Обновить место захоронения
```http
POST /api/memorial-pages/:pageId/burial-location
Authorization: Bearer <token>
Content-Type: application/json

{
  "address": "Москва, Новодевичье кладбище",
  "description": "Участок 2, ряд 5",
  "instructions": "От главного входа прямо 100м"
}
```

**Ответ:**
```json
{
  "success": true,
  "data": {
    "id": "...",
    "memorialPageId": "...",
    "address": "Москва, Новодевичье кладбище",
    "latitude": 55.726667,
    "longitude": 37.555556,
    "description": "Участок 2, ряд 5",
    "instructions": "От главного входа прямо 100м"
  }
}
```

### Получить место захоронения
```http
GET /api/memorial-pages/:pageId/burial-location
```

### Геокодирование адреса
```http
POST /api/geocode
Content-Type: application/json

{
  "address": "Москва, Красная площадь"
}
```

**Ответ:**
```json
{
  "success": true,
  "data": {
    "latitude": 55.753544,
    "longitude": 37.621211,
    "formattedAddress": "Россия, Москва, Красная площадь"
  }
}
```

## Без API ключей

Если у вас нет API ключей, можно:

### Вариант 1: Ввод координат вручную
Добавить поля для ручного ввода:
- Широта (latitude): -90 до 90
- Долгота (longitude): -180 до 180

### Вариант 2: Использовать другой сервис карт
- OpenStreetMap (бесплатно)
- Google Maps (платно)
- Mapbox (бесплатный tier)

### Вариант 3: Только текстовое описание
Показывать только адрес и инструкции без карты.

## Стоимость использования

### Яндекс.Карты
- **Бесплатно:** до 25,000 запросов в день
- **Платно:** от 0.002₽ за запрос сверх лимита

### Яндекс.Геокодер
- **Бесплатно:** до 1,000 запросов в день
- **Платно:** от 0.02₽ за запрос сверх лимита

Для небольшого проекта бесплатного лимита достаточно.

## Альтернативы Яндекс.Картам

### 1. OpenStreetMap + Leaflet
**Плюсы:**
- Полностью бесплатно
- Нет лимитов
- Открытый исходный код

**Минусы:**
- Менее детальные карты России
- Нужно настраивать самостоятельно

### 2. Google Maps
**Плюсы:**
- Отличное качество карт
- Много функций

**Минусы:**
- Платно (требуется кредитная карта)
- $200 бесплатных кредитов в месяц

### 3. 2GIS
**Плюсы:**
- Хорошие карты России
- Бесплатный API

**Минусы:**
- Ограниченная география
- Меньше функций

## Troubleshooting

### Карта не загружается
**Проблема:** Белый экран вместо карты  
**Решение:** Проверьте `NEXT_PUBLIC_YANDEX_MAPS_API_KEY` в `.env.local`

### Координаты не определяются
**Проблема:** Адрес не преобразуется в координаты  
**Решение:** Проверьте `YANDEX_GEOCODER_API_KEY` в `backend/.env`

### Ошибка "API key invalid"
**Проблема:** Неверный ключ  
**Решение:** Проверьте правильность ключа, убедитесь что он активирован

### Превышен лимит запросов
**Проблема:** "Quota exceeded"  
**Решение:** Подождите до следующего дня или перейдите на платный план
