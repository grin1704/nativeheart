# 🚀 Скрипты для разработки Memorial Pages

Набор удобных скриптов для автоматизации запуска, остановки и мониторинга проекта.

## 📋 Доступные скрипты

### 🚀 `./scripts/dev-start.sh`
**Основной скрипт для запуска проекта**
- Автоматически очищает занятые порты 3000 и 3001
- Проверяет и устанавливает зависимости при необходимости
- Запускает backend и frontend одновременно
- Показывает полезную информацию (URL, админ данные)
- Работает до нажатия Ctrl+C

### 🛑 `./scripts/dev-stop.sh`
**Остановка всех сервисов**
- Корректно останавливает backend и frontend
- Очищает PID файлы
- Принудительно освобождает порты при необходимости

### 🔄 `./scripts/dev-restart.sh`
**Перезапуск проекта**
- Останавливает все сервисы
- Ждет 2 секунды
- Запускает заново

### 📊 `./scripts/dev-status.sh`
**Проверка статуса сервисов**
- Показывает статус портов 3000 и 3001
- Проверяет HTTP доступность
- Выводит полезные URL и команды

### 🧹 `./scripts/port-cleanup.sh`
**Экстренная очистка портов**
- Интерактивно показывает занятые порты
- Позволяет выборочно остановить процессы
- Полезно при зависших процессах

## 🎯 Быстрый старт

```bash
# Запуск проекта (рекомендуемый способ)
./scripts/dev-start.sh

# В другом терминале - проверка статуса
./scripts/dev-status.sh

# Остановка
./scripts/dev-stop.sh
```

## 🔧 Решение проблем

### Порты заняты
```bash
# Экстренная очистка
./scripts/port-cleanup.sh

# Или принудительная остановка
./scripts/dev-stop.sh
```

### Зависшие процессы
```bash
# Проверка статуса
./scripts/dev-status.sh

# Принудительная остановка
./scripts/dev-stop.sh

# Перезапуск
./scripts/dev-restart.sh
```

### Проблемы с зависимостями
```bash
# Скрипт автоматически проверит и установит зависимости
./scripts/dev-start.sh
```

## 💡 Советы

1. **Всегда используйте `dev-start.sh`** вместо ручного запуска
2. **Проверяйте статус** с помощью `dev-status.sh`
3. **При проблемах** сначала попробуйте `dev-restart.sh`
4. **Для экстренных случаев** используйте `port-cleanup.sh`

## 🚨 Troubleshooting

| Проблема | Решение |
|----------|---------|
| Порт занят | `./scripts/port-cleanup.sh` |
| Сервис не отвечает | `./scripts/dev-restart.sh` |
| Зависимости устарели | Удалите `node_modules` и запустите `dev-start.sh` |
| Процессы зависли | `./scripts/dev-stop.sh` затем `dev-start.sh` |
