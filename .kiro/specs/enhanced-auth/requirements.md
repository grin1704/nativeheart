# Требования: Расширенная авторизация

## Обзор

Расширение существующей системы авторизации дополнительными функциями:
- Верификация email
- Восстановление пароля
- OAuth авторизация (VK, Yandex)

## Текущее состояние

### Что уже работает ✅

- Регистрация пользователей (email + password)
- Авторизация с JWT токенами
- Управление подписками
- Защищенные маршруты

### Что нужно добавить 🎯

1. **Верификация email**
   - Отправка письма с токеном при регистрации
   - Подтверждение email по ссылке
   - Повторная отправка письма
   - Блокировка некоторых функций до верификации

2. **Восстановление пароля**
   - Запрос на восстановление (отправка письма)
   - Проверка токена восстановления
   - Установка нового пароля
   - Истечение токена через 1 час

3. **OAuth авторизация**
   - Вход через ВКонтакте
   - Вход через Яндекс
   - Автоматическое создание аккаунта при первом входе
   - Связывание OAuth с существующим аккаунтом

## Технические требования

### API Endpoints

#### Верификация email

- `POST /api/auth/verify-email` - подтверждение email
  - Body: `{ token: string }`
  - Response: `{ message: string }`

- `POST /api/auth/resend-verification` - повторная отправка письма
  - Body: `{ email: string }`
  - Response: `{ message: string }`

#### Восстановление пароля

- `POST /api/auth/request-password-reset` - запрос на восстановление
  - Body: `{ email: string }`
  - Response: `{ message: string }`

- `POST /api/auth/reset-password` - установка нового пароля
  - Body: `{ token: string, password: string }`
  - Response: `{ message: string }`

#### OAuth

- `GET /api/auth/vk/url` - получить URL для авторизации ВК
  - Response: `{ authUrl: string }`

- `POST /api/auth/vk/callback` - обработка callback от ВК
  - Body: `{ code: string }`
  - Response: `{ user, token }`

- `GET /api/auth/yandex/url` - получить URL для авторизации Яндекс
  - Response: `{ authUrl: string }`

- `POST /api/auth/yandex/callback` - обработка callback от Яндекс
  - Body: `{ code: string }`
  - Response: `{ user, token }`

### Frontend страницы

- `/auth/verify-email` - страница подтверждения email
- `/auth/forgot-password` - форма запроса восстановления пароля
- `/auth/reset-password` - форма установки нового пароля
- `/auth/vk/callback` - обработка callback от ВК
- `/auth/yandex/callback` - обработка callback от Яндекс

## Безопасность

1. **Токены**
   - Генерировать криптографически стойкие токены (crypto.randomBytes)
   - Хранить токены в хешированном виде (опционально)
   - Проверять срок действия токенов

2. **OAuth**
   - Проверять state параметр для защиты от CSRF
   - Валидировать redirect_uri
   - Не доверять данным от OAuth провайдера без проверки

3. **Email**
   - Не раскрывать существование email в системе
   - Использовать rate limiting для отправки писем
   - Логировать все попытки восстановления пароля

## Ограничения

1. **Rate Limiting**
   - Верификация email: 3 попытки в час на email
   - Восстановление пароля: 3 попытки в час на email
   - OAuth: 10 попыток в час на IP

2. **Валидация**
   - Email должен быть валидным
   - Пароль минимум 6 символов
   - Токены должны быть действительными

## Приоритеты

1. **Высокий приоритет**
   - Восстановление пароля (критично для пользователей)
   - Верификация email (безопасность)

2. **Средний приоритет**
   - OAuth ВКонтакте (удобство)
   - OAuth Яндекс (удобство)

## Риски и предосторожности

⚠️ **ВАЖНО:** При реализации этой задачи в прошлый раз возникли критические проблемы:

1. **Проблема с Prisma схемой**
   - Команда `prisma db pull` испортила схему
   - Prisma начал генерировать snake_case типы вместо camelCase
   - Backend перестал компилироваться

2. **Решение**
   - НЕ использовать `prisma db pull`
   - Использовать только `prisma migrate dev` для изменений схемы
   - Всегда делать бэкап перед изменениями

3. **Бэкап**
   - Перед началом работы создан полный бэкап: `backups/backup_20251115_235846`
   - Восстановление: `./backups/backup_20251115_235846/restore.sh`
   - Инструкции: `QUICK_RESTORE.md`
