# Timeline Feature - Требования

## Обзор
Добавление функционала "Хронология жизни" (Timeline) на памятные страницы. Позволяет отображать ключевые события жизни человека в хронологическом порядке с указанием года и описания события.

## Функциональные требования

### 1. Модель данных
- Таблица `TimelineEvent` с полями:
  - `id` - уникальный идентификатор
  - `memorialPageId` - связь с памятной страницей
  - `year` - год события (число)
  - `description` - описание события (текст, до 500 символов)
  - `order` - порядок отображения
  - `createdAt`, `updatedAt` - временные метки

### 2. Backend API
- `GET /api/memorial-pages/:id/timeline` - получить все события
- `POST /api/memorial-pages/:id/timeline` - создать событие
- `PUT /api/memorial-pages/:id/timeline/:eventId` - обновить событие
- `DELETE /api/memorial-pages/:id/timeline/:eventId` - удалить событие
- `PUT /api/memorial-pages/:id/timeline/reorder` - изменить порядок событий

### 3. Редактор (Dashboard)
Компонент `TimelineEditor`:
- Список всех событий с возможностью drag-and-drop для изменения порядка
- Форма добавления нового события (год + описание)
- Редактирование существующих событий
- Удаление событий
- Валидация: год должен быть числом, описание обязательно

### 4. Публичное отображение
Компонент `MemorialTimeline`:
- Вертикальная линия с кружками (как на референсе)
- Год слева, описание справа в карточке
- События отсортированы по году (от раннего к позднему)
- Адаптивный дизайн для мобильных устройств

## Визуальный дизайн
- Стиль похож на компонент Memories
- Бежевый фон карточек
- Вертикальная линия с кружками-маркерами
- Год отображается крупным шрифтом слева
- Описание в светлой карточке справа

## Ограничения
- Максимум 50 событий на одну страницу
- Год: от 1800 до текущего года + 10 лет
- Описание: максимум 500 символов

## Приоритет
Средний - новый функционал, не критичный для существующих пользователей
