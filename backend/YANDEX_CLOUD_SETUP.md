# Настройка Yandex Cloud Object Storage

## Обзор

Для работы системы управления медиафайлами необходимо настроить Yandex Cloud Object Storage. Этот документ содержит пошаговую инструкцию по созданию bucket'а и получению ключей доступа.

## Шаг 1: Создание аккаунта Yandex Cloud

1. Перейдите на [cloud.yandex.ru](https://cloud.yandex.ru)
2. Войдите в систему или создайте новый аккаунт
3. Активируйте пробный период (если доступен)

## Шаг 2: Создание сервисного аккаунта

1. В консоли Yandex Cloud перейдите в раздел **IAM**
2. Выберите **Сервисные аккаунты**
3. Нажмите **Создать сервисный аккаунт**
4. Укажите имя: `memorial-pages-storage`
5. Добавьте роли:
   - `storage.editor` - для управления объектами
   - `storage.viewer` - для чтения объектов
6. Нажмите **Создать**

## Шаг 3: Создание статических ключей доступа

1. Откройте созданный сервисный аккаунт
2. Перейдите на вкладку **Ключи доступа**
3. Нажмите **Создать новый ключ** → **Создать статический ключ доступа**
4. Сохраните:
   - **Идентификатор ключа** (Access Key ID)
   - **Секретный ключ** (Secret Access Key)

⚠️ **Важно**: Секретный ключ показывается только один раз. Обязательно сохраните его в безопасном месте.

## Шаг 4: Создание Object Storage bucket

1. В консоли Yandex Cloud перейдите в **Object Storage**
2. Нажмите **Создать бакет**
3. Настройте параметры:
   - **Имя**: `memorial-pages-storage` (или другое уникальное имя)
   - **Класс хранилища**: Стандартное
   - **Доступ на чтение**: Публичный (для доступа к загруженным файлам)
   - **Доступ к списку объектов**: Ограниченный
4. Нажмите **Создать бакет**

## Шаг 5: Настройка CORS (опционально)

Если планируется загрузка файлов напрямую из браузера:

1. Откройте созданный bucket
2. Перейдите на вкладку **CORS**
3. Добавьте правило:

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT", "POST", "DELETE"],
    "AllowedOrigins": ["http://localhost:3000", "https://yourdomain.com"],
    "ExposeHeaders": ["ETag"],
    "MaxAgeSeconds": 3000
  }
]
```

## Шаг 6: Обновление переменных окружения

Добавьте полученные данные в файл `.env`:

```bash
# Yandex Cloud Object Storage
YANDEX_CLOUD_ACCESS_KEY_ID=YCAJExxxxxxxxxxxxxxxxx
YANDEX_CLOUD_SECRET_ACCESS_KEY=YCOxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
YANDEX_CLOUD_BUCKET_NAME=memorial-pages-storage
YANDEX_CLOUD_REGION=ru-central1
```

## Шаг 7: Проверка настройки

После настройки можно проверить подключение:

```bash
# Установить зависимости
npm install aws-sdk sharp mime-types

# Запустить тест
node test-media.js
```

## Структура bucket'а

После настройки файлы будут организованы следующим образом:

```
memorial-pages-storage/
├── images/
│   ├── 2024/
│   │   ├── 01/
│   │   └── 02/
│   └── 2023/
├── videos/
│   ├── 2024/
│   │   ├── 01/
│   │   └── 02/
│   └── 2023/
└── thumbnails/
    ├── 2024/
    │   ├── 01/
    │   └── 02/
    └── 2023/
```

## Стоимость

Примерная стоимость хранения (на январь 2024):

- **Хранение**: ~2.5 ₽ за ГБ в месяц
- **Исходящий трафик**: ~3 ₽ за ГБ
- **API-запросы**: ~0.4 ₽ за 1000 запросов

Для небольшого проекта стоимость будет минимальной.

## Безопасность

### Рекомендации:

1. **Ограничьте доступ**: Используйте минимально необходимые права
2. **Ротация ключей**: Периодически обновляйте статические ключи
3. **Мониторинг**: Настройте уведомления о необычной активности
4. **Backup**: Регулярно создавайте резервные копии важных файлов

### Настройка политики доступа:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::memorial-pages-storage/*"
    }
  ]
}
```

## Альтернативные варианты

Если Yandex Cloud недоступен, можно использовать:

1. **AWS S3** - изменить endpoint в коде
2. **VK Cloud** - аналогично S3
3. **Локальное хранилище** - для разработки

## Поддержка

При возникновении проблем:

1. Проверьте [документацию Yandex Cloud](https://cloud.yandex.ru/docs/storage/)
2. Убедитесь в правильности ключей доступа
3. Проверьте настройки CORS для веб-загрузки
4. Проверьте квоты и лимиты аккаунта

## Готовые команды для быстрой настройки

```bash
# 1. Установить зависимости
npm install aws-sdk sharp mime-types

# 2. Создать .env файл (если не существует)
cp .env.example .env

# 3. Отредактировать .env файл с вашими ключами
# YANDEX_CLOUD_ACCESS_KEY_ID=ваш_ключ
# YANDEX_CLOUD_SECRET_ACCESS_KEY=ваш_секретный_ключ
# YANDEX_CLOUD_BUCKET_NAME=ваш_bucket

# 4. Проверить подключение
node test-media.js
```

После выполнения этих шагов система управления медиафайлами будет полностью готова к работе!
