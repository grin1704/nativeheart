# 🚀 Быстрый старт: Подключение Yandex Cloud

## Что нужно сделать

### 1️⃣ Создать аккаунт и bucket в Yandex Cloud

**Перейдите на:** https://cloud.yandex.ru

1. **Войдите** или зарегистрируйтесь
2. **Активируйте пробный период** (если доступен - дают бесплатные 4000₽)

### 2️⃣ Создать сервисный аккаунт

1. В консоли → **IAM** → **Сервисные аккаунты**
2. **Создать сервисный аккаунт**
3. Имя: `memorial-pages-storage`
4. Роли:
   - ✅ `storage.editor`
   - ✅ `storage.viewer`
5. **Создать**

### 3️⃣ Получить ключи доступа

1. Откройте созданный сервисный аккаунт
2. **Ключи доступа** → **Создать новый ключ** → **Создать статический ключ доступа**
3. **СОХРАНИТЕ** (показывается только один раз!):
   - 📋 **Идентификатор ключа** (Access Key ID)
   - 🔑 **Секретный ключ** (Secret Access Key)

### 4️⃣ Создать bucket

1. **Object Storage** → **Создать бакет**
2. Настройки:
   - **Имя**: `memorial-pages-prod` (или любое уникальное)
   - **Класс хранилища**: Стандартное
   - **Доступ на чтение**: ✅ **Публичный** (важно!)
   - **Доступ к списку объектов**: Ограниченный
3. **Создать бакет**

### 5️⃣ Настроить CORS (для загрузки из браузера)

1. Откройте bucket → вкладка **CORS**
2. Добавьте правило:

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT", "POST", "DELETE"],
    "AllowedOrigins": ["http://localhost:3000", "http://localhost:3001"],
    "ExposeHeaders": ["ETag"],
    "MaxAgeSeconds": 3000
  }
]
```

### 6️⃣ Добавить ключи в .env

Откройте `backend/.env` и добавьте:

```bash
# Yandex Cloud Object Storage
YANDEX_CLOUD_ACCESS_KEY_ID=YCAJExxxxxxxxxxxxxxxxx
YANDEX_CLOUD_SECRET_ACCESS_KEY=YCOxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
YANDEX_CLOUD_BUCKET_NAME=memorial-pages-prod
YANDEX_CLOUD_REGION=ru-central1

# Переключить на Yandex Cloud (вместо локального хранилища)
STORAGE_TYPE=yandex
```

### 7️⃣ Установить зависимости

```bash
cd backend
npm install aws-sdk sharp mime-types
```

### 8️⃣ Перезапустить бэкенд

```bash
# Остановить текущий процесс (Ctrl+C)
# Запустить снова
npm run dev
```

## ✅ Проверка

После настройки попробуйте загрузить фото в админке или на странице памятника. Файлы должны загружаться в Yandex Cloud!

## 📊 Мониторинг

Проверить загруженные файлы:
1. Yandex Cloud → Object Storage → Ваш bucket
2. Увидите структуру папок: `images/`, `videos/`, `thumbnails/`

## 💰 Стоимость

**Примерно:**
- Хранение: ~2.5₽ за ГБ/месяц
- Трафик: ~3₽ за ГБ
- Для небольшого проекта: ~100-300₽/месяц

**Пробный период:** 4000₽ бесплатно на 60 дней

## 🔧 Переключение между локальным и облачным хранилищем

В `backend/.env`:

```bash
# Локальное хранилище (для разработки)
STORAGE_TYPE=local

# Yandex Cloud (для продакшена)
STORAGE_TYPE=yandex
```

## ❓ Проблемы?

### Ошибка "Access Denied"
- Проверьте правильность ключей
- Убедитесь, что bucket публичный
- Проверьте роли сервисного аккаунта

### Ошибка CORS
- Добавьте правило CORS в bucket
- Укажите правильные origins (localhost:3000, localhost:3001)

### Файлы не загружаются
- Проверьте `STORAGE_TYPE=yandex` в .env
- Перезапустите бэкенд
- Проверьте логи в консоли
