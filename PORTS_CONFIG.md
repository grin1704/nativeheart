# 🔧 Конфигурация портов

## 📌 ФИКСИРОВАННЫЕ ПОРТЫ - НЕ МЕНЯТЬ!

### Фронтенд (Next.js)
- **Порт:** `3000`
- **URL:** `http://localhost:3000`
- **Конфигурация:** `frontend/package.json` - скрипт `dev`

### Бэкенд (Node.js/Express)
- **Порт:** `3001` 
- **URL:** `http://localhost:3001`
- **Конфигурация:** `backend/.env` - переменная `PORT=3001`

## 🚫 ВАЖНО: НЕ ИЗМЕНЯТЬ ПОРТЫ!

Эти порты жестко прописаны в:
- ✅ `frontend/.env.local` - `NEXT_PUBLIC_API_URL=http://localhost:3001`
- ✅ `backend/.env` - `PORT=3001`
- ✅ `backend/.env` - `FRONTEND_URL=http://localhost:3000`
- ✅ Все API роуты в `frontend/src/app/api/`
- ✅ Все компоненты фронтенда

## 📋 Проверка портов

### Проверить, что порты свободны:
```bash
# Проверить порт 3000 (фронтенд)
lsof -i :3000

# Проверить порт 3001 (бэкенд)  
lsof -i :3001
```

### Освободить порты при необходимости:
```bash
# Остановить процесс на порту 3000
kill -9 $(lsof -ti:3000)

# Остановить процесс на порту 3001
kill -9 $(lsof -ti:3001)
```

## 🚀 Запуск сервисов

### Бэкенд:
```bash
cd backend
npm run dev
# Должен запуститься на http://localhost:3001
```

### Фронтенд:
```bash
cd frontend  
npm run dev
# Должен запуститься на http://localhost:3000
```

## 🔗 Архитектура подключений

```
Браузер (localhost:3000)
    ↓ API запросы к /api/*
Next.js API Routes (localhost:3000/api/*)
    ↓ Проксирование к бэкенду
Express Backend (localhost:3001/api/*)
    ↓ Запросы к БД
PostgreSQL Database
```

## ⚠️ Если что-то не работает

1. **Проверьте порты:** Убедитесь, что оба сервиса запущены на правильных портах
2. **Проверьте .env файлы:** Убедитесь, что переменные окружения правильные
3. **Перезапустите сервисы:** Остановите и запустите заново оба сервиса
4. **Проверьте логи:** Смотрите вывод в терминалах на наличие ошибок

## 🎯 Итоговая конфигурация

- **Фронтенд:** ВСЕГДА на порту 3000
- **Бэкенд:** ВСЕГДА на порту 3001  
- **API проксирование:** Через Next.js API routes
- **Никаких прямых обращений к localhost:3001 из браузера**
